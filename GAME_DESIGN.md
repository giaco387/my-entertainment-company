# 我的娱乐公司 — 开发者文档

## 一、项目架构

```
index.html          主文件：全部 HTML / CSS / JS（约 2200 行）
artists.js          艺人数据池：POOL（8人）、DIRECTIONS（4个方向）
events.js           活动数据：EVENTS（9个主动活动）、MONTHLY_EVENTS（被动随机）
content.js          内容发行：CONTENT_TYPES（4种）、CONTENT_TITLES
achievements.js     成就定义：ACHIEVEMENTS（18个）
market.js           市场趋势：TREND_POOL（12条趋势）
agents.js           经纪人池：AGENT_POOL（5人）
```

无任何外部依赖，直接用浏览器打开 index.html 即可运行。

## 二、全局状态 G

所有游戏数据挂在 `G` 对象上，通过 `saveGame()` / `loadGame()` 序列化到 `localStorage`（key: `mec_save`）。

```javascript
G = {
  money: 800,            // 资金（万元）
  fame: 12,              // 公司声望
  month: 1,              // 当前月份（游戏时间轴）
  artists: [],           // 艺人数组（见下方艺人结构）
  buildings: { ... },    // 10个设施对象（lv / maxLv / cost / ...）
  releases: [],          // 已发行内容列表
  pendingEvents: [],     // 进行中活动队列
  pendingReleases: [],   // 制作中内容队列
  pendingBuilds: [],     // 施工中设施队列
  agents: [],            // 在职经纪人 id 列表
  rivals: [],            // 竞争对手数组
  activeTrends: [],      // 当前生效的市场趋势
  achievements: [],      // 已解锁成就 id 列表
  unlockedTabs: [],      // 已解锁的导航页签 id 列表
  monthlyHistory: [],    // 近24个月收支历史（用于图表）
  log: [],               // 收支流水（最多50条）
  lastEvents: [],        // 本月发生的事件（用于总览展示）
  usedEvents: [],        // 本月已参加的活动下标
  usedEventArtists: [],  // 本月已出动的艺人下标
  completedEventNames: [],// 历史上完成过的活动名称（用于成就）
  buildingActions: {},   // 设施本月行动记录 {k: month}
  streamingDeal: false,  // 是否签约流媒体平台
  lastAwardMonth: 0,     // 上次颁奖典礼触发的月份
}
```

**艺人对象结构：**
```javascript
{
  name, abbr, color, tc,          // 基础信息（来自 POOL）
  singing, dance, acting, rap,    // 四维属性（0-99）
  rarity,                         // 稀有度：普通 / 潜力股 / 天才
  status,          // 训练中 | 已出道 | 工作中 | 休息中
  fans,            // 粉丝量（万）
  pr,              // 舆论值（0-100，默认60）
  energy,          // 精力值（0-100）
  direction,       // 发展方向：歌手 | 演员 | 主持人 | 全能
  alias,           // 艺名（可选）
  contractEnd,     // 合同到期月份
  totalIncome,     // 累计收入
  totalCost,       // 累计成本
  _hadCrisis,      // 是否经历过舆论危机（用于「浴火重生」成就）
}
```

## 三、核心游戏循环

**时间驱动：** `startDayTick()` 每 1.5 秒推进一天，一个月约 45 秒现实时间。日期走完触发 `nextMonth()`。

**`nextMonth()` 执行顺序：**

```
1.  G.month++，重置月度行动记录（usedEvents / usedEventArtists / lastEvents）
2.  updateTrends()        —— 更新市场趋势（老趋势-1月，随机补充新趋势）
3.  扣经纪人薪资
4.  处理 pendingEvents    —— 完成的活动发放奖励，apply() 执行效果
5.  处理 pendingBuilds    —— 完成的施工升级 b.lv = targetLv
6.  处理 pendingReleases  —— 内容制作完成，移入 releases 正式发行
7.  直播间 / 商品企划部 —— 计算被动收入
8.  遍历 releases         —— 按时间衰减计算内容发行收益（mult: 1.5→1.0→0.6→0.2）
9.  遍历 artists          —— 训练成长 / 工作收入 / 休息恢复 / 精力消耗
    ├ 精力≤0 → 强制休息，加入 exhausted[]
    └ PR<60 → 自然恢复（+2/月）；PR<40 → 掉粉（×0.95/月）
10. G.money += income - cost；G.fame += floor(income/25+1)
11. triggerMonthlyEvents() —— 随机被动事件（行业/公司/艺人）+ 颁奖典礼判断
12. exhausted 推入 lastEvents
13. doRivalAction()（每3个月）
14. PR经纪人自动给PR最低艺人+10
15. 合同到期检查（到期当月推送提醒）
16. 更新UI，存档，checkAchievements()
```

## 四、各系统详解

### 4.1 艺人系统

**出道条件：** 训练中 singing > 75 且 dance > 65 时自动出道。

**收入公式（工作中）：**
```
base = (singing + dance) / 12 + random(0~8)
方向加成：歌手+唱≥70 ×1.25；演员+演≥70 ×1.25；全能 ×1.1
MV拍摄间有 Lv ×1.3
PR乘数：80+→1.3 / 60-79→1.0 / 40-59→0.7 / <40→0.3
经济下行趋势 ×0.75；精力≤20 ×0.6
```

**精力机制：**
- 训练中：-10/月；工作中：-15/月；休息中：+30/月（健康管理中心每级+3）
- 精力归零 → 强制切换到休息中

**PR舆论：**
- 自然回复：低于60时每月+2，上限60
- PR < 40 视为舆论危机，此状态持续会掉粉（×0.95/月）

### 4.2 设施系统

| 设施 | 最高等级 | 核心效果 |
|------|---------|---------|
| 直播间 | 8 | 月收入 lv×15 万 |
| 训练室 | 8 | 训练效率 +20%/级 |
| 录音室 | 8 | 解锁音乐类内容发行 |
| 粉丝见面厅 | 6 | 解锁粉丝运营操作 |
| MV拍摄间 | 6 | 内容/工作收益 ×1.3 |
| 经纪办公室 | 6 | 签约上限 = 2 + lv×2 |
| 周边商品部 | 6 | 被动收益 = 总粉丝/100×lv×2 |
| 媒体中心 | 6 | 艺人坏事件概率 ×(1-lv×8%) |
| 健康管理中心 | 6 | 休息时额外恢复精力 lv×3 |
| 法务部 | 6 | 发声明修复PR；降低续签费 |

**施工时间：** 升至 Lv1=1个月，Lv2-3=2个月，Lv4-5=3个月，Lv6+=4个月。施工期间仍可使用，升级效果完工后生效。

**每月设施行动：** 各设施在"进入室内"后可使用一次主动行动（如直播创收、安排训练加速、做粉丝见面会等），每月每设施限一次，由 `G.buildingActions[k]` 记录。

### 4.3 内容发行

```
制作（N个月）→ 自动发布 → 按时间衰减产生收益（最多持续 duration 个月）

收益衰减乘数：首月×1.5，次月×1.0，第3月×0.6，之后×0.2
流媒体合约（50万，录音室Lv2+声望≥40）：单曲/专辑收益永久×1.5
金牌制作人经纪人：所有内容+30%
```

| 类型 | 成本 | 制作期 | 热度期 | 解锁条件 |
|------|------|--------|--------|---------|
| 数字单曲 | 20万 | 1月 | 2月 | 录音室Lv1 |
| 实体专辑 | 80万 | 2月 | 4月 | 录音室Lv3 |
| MV | 50万 | 1月 | 3月 | MV拍摄间Lv1 |
| 纪录片 | 40万 | 2月 | 2月 | 任意设施Lv2 |

### 4.4 活动赛事

每月可参加任意多个活动，但**同一活动每月限参加一次**，且**每位艺人每月只能出动一次**。

参加后进入 `pendingEvents` 队列，`duration` 个月后结算奖励并执行 `apply()` 效果（粉丝增长、PR提升等）。

市场趋势叠加：`idol_boom` 对选秀×1.5，`concert_craze` 对演唱会×1.5，`economy_down` 对所有活动收益×0.75。

### 4.5 市场趋势

每月：现有趋势剩余月份-1，活跃数<3时35%概率随机补充一条新趋势，最多同时3条。趋势效果全部**自动生效**，无需玩家干预。正面趋势2-3个月，负面1-2个月。

### 4.6 经纪人

首次签约需预付6个月薪资，之后每月自动扣除。

| 经纪人 | 月薪 | 效果 |
|--------|------|------|
| 商务经纪人 | 15万 | 代言/合作活动收益+40% |
| 公关专家 | 20万 | 每月自动给PR最低的艺人+10 |
| 资深星探 | 12万 | 招募时显示6人候选（默认4人） |
| 金牌制作人 | 28万 | 所有内容发行收益+30% |
| 训练总监 | 18万 | 训练效率额外+40% |

### 4.7 竞争对手

每3个月触发一次，随机选择：
- 30%：让玩家某位艺人掉粉7%
- 20%：让玩家公司声望-4%
- 50%：无动作

玩家可花50万压制（对方声望-15%）或30万合作（自家声望+15）。

### 4.8 被动随机事件（triggerMonthlyEvents）

- **行业事件**：每3个月触发1个（好/坏各有5条）
- **公司事件**：每月15%概率触发1个
- **艺人事件**：每位艺人独立10%概率触发（媒体中心每级降低8%）；存在scandal趋势时提升至20%
- **颁奖典礼**：声望≥150，6个月冷却，概率随声望增长（最高45%）

### 4.9 公司段位

| 段位 | 声望要求 |
|------|---------|
| ⭐ 初创公司 | 0–199 |
| ⭐⭐ 新兴公司 | 200–799 |
| ⭐⭐⭐ 知名公司 | 800–2499 |
| ⭐⭐⭐⭐ 顶级公司 | 2500–7999 |
| 🌟 娱乐帝国 | 8000+ |

### 4.10 功能渐进解锁

| 触发条件 | 解锁功能 |
|---------|---------|
| 始终可用 | 总览、艺人管理、设施建设 |
| 签约≥1人 或 第3个月 | 内容发行、财务报表 |
| 有艺人出道 或 第6个月 | 活动赛事、市场动态 |
| 解锁≥1个成就 或 第3个月 | 成就中心 |
| 声望≥80 或 第9个月 | 经纪人团队 |
| 声望≥150 或 第12个月 | 竞争对手 |

# 普通玩家游玩路线

## 第1-3个月：打基础

**目标：招人 + 建设施 + 活下去**

1. 进入游戏，资金 800 万
2. 先去**设施建设**，优先建造「**直播间**」（80万），建好后每月有稳定收入（约15万）
3. 再去**艺人管理 → 招募练习生**，选一位，推荐选合同2年（标准费）
4. 练习生签约后，切到「**训练安排**」标签，状态保持「训练中」，等待属性成长
5. 训练同时可以升级「**训练室**」提高效率
6. 这几个月耐心等待，总览面板会提示「有练习生训练中」

> 第3个月保底解锁「内容发行」和「财务报表」，但此时艺人可能还没出道，先别急着发行

---

## 第4-6个月：第一位艺人出道

**目标：出道 → 开始接工作 → 发第一首歌**

1. 当艺人 singing > 75 且 dance > 65，**自动出道**，并弹出提示
2. 出道后立刻进「训练安排」，把状态切换成「**工作中**」开始赚钱
3. 此时「**活动赛事**」和「**市场动态**」解锁
4. 去「**内容发行**」发一首数字单曲（20万，1个月后上线）
5. 如果经济够用，建造「**录音室 Lv.1**」以支持后续更多内容类型
6. 注意艺人精力：工作状态每月消耗15点，精力低于20时收益打折，耗尽则强制休息

> 留意总览的「本月事件」，可能有随机危机（坏事件）影响PR，PR跌到40以下会开始掉粉

---

## 第7-12个月：扩张规模

**目标：多签人 + 参加活动 + 声望冲80解锁经纪人**

1. 升级「**经纪办公室**」（默认容量4人），招募第2、3位艺人
2. 多位艺人出道后，去「**活动赛事**」参加「演唱会巡演」「新人选秀综艺」等大型活动
3. 声望接近80时，「**经纪人团队**」解锁，优先考虑：
   - **训练总监**（18万/月）：训练效率+40%，加速出道
   - **公关专家**（20万/月）：自动维护PR，省心
4. 同期去「**内容发行**」签约**流媒体平台**（50万一次性，录音室Lv2解锁），单曲/专辑收益永久×1.5
5. 给出道艺人设置**发展方向**（歌手/演员/全能），激活方向专属加成

---

## 第13-24个月：深度运营

**目标：声望冲150解锁竞争对手 + 收割成就 + 建设完善**

1. 声望≥150 解锁「**竞争对手**」，每3个月他们会干扰你，留点资金准备应对
2. 建造「**粉丝见面厅**」解锁粉丝运营操作，给高粉艺人做粉丝活动放大人气
3. 注意**合同到期提醒**，到期当月去「艺人管理→合同」续签，法务部Lv越高续签费越低
4. 发行「实体专辑」（需录音室Lv3），收益最高的内容类型
5. 参加「**海外巡演**」（需声望≥250，2位出道艺人），单次奖励最丰厚
6. 解锁到「🌟 娱乐帝国」（声望8000）是终极目标，没有硬性时间限制，慢慢玩

## 关键优先级备忘

```
资金紧张时：  直播间 > 训练室 > 经纪办公室
声望想快升：  参加活动 > 发行内容 > 工作中艺人自然积累
艺人快出道：  训练总监 + 训练室升级 + 让艺人专注训练
PR危机处理：  法务部「发声明」 或 切换休息状态自然恢复
```
